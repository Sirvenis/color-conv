CSS Color Formats Explained: HEX, RGB, HSL, HWB, and More

Published: April 2026 | Reading time: 7 min

CSS supports a growing number of color formats. Beyond the classic HEX, RGB, and HSL, modern CSS adds HWB, LCH, OKLCH, and color-mix(). Here's what each does and when to use it.

Classic Formats

HEX

The most widely used format. #RRGGBB or #RGB (shorthand). Supports alpha with 8 digits: #RRGGBBAA.

color: #e94560;

RGB / RGBA

Red, Green, Blue values from 0-255. RGBA adds alpha (0-1). Native to CSS, slightly faster parsing.

color: rgb(233, 69, 96);

HSL / HSLA

Hue (0-360), Saturation (0-100%), Lightness (0-100%). Best for creating color variations.

color: hsl(348, 78%, 59%);

Modern CSS Color Formats

HWB

Hue, Whiteness, Blackness. Similar to HSL but more intuitive for some designers. Values are 0-100%.

color: hwb(348 9% 19%);

LCH and OKLCH

Perceptually uniform color spaces — colors that look equally bright to humans actually are equally bright. OKLCH is the modern recommendation, fixing some issues with LCH.

color: oklch(60% 0.2 20);

These are especially powerful for design systems because adjusting lightness produces truly perceptually even gradients.

color-mix()

Mixes two colors in a specified color space. Incredibly useful for generating hover states, tints, and shades programmatically.

color: color-mix(in oklch, #e94560 70%, white);

When to Use Which Format

  • HEX: Brand colors, design tokens, static values
  • RGB: Dynamic colors via JavaScript, opacity variations
  • HSL: Theme systems, creating lighter/darker variants
  • OKLCH: Design systems needing perceptual uniformity
  • color-mix(): Programmatic color generation, hover states

Conclusion

HEX and RGB remain the workhorses, but OKLCH and color-mix() represent the future of CSS color. Start experimenting with modern formats for new projects while keeping HEX as your fallback.
